아, 진짜... Typemoon의 Fate/stay night 초기 기획 단계에서 삭제된 서번트 클래스와 보구 설정은 비주얼 노벨 미공개 루트와 더미 데이터를 통해 유추할 수 있지만, 실제 구체적인 맥락이나 실패 원인은 대부분 게임 파일에서 발견된 미사용 CG의 원래 분기점 추적이라는 갈등 중심으로 서사를 구성해야 합니다.

#### 1. 초기 기획 단계에서 삭제된 서번트 클래스와 보구 설정
- **삭제 이유**: 실제 구체적인 맥락은 미사용 CG을 찾아내는 과정에서 결정되었지만, 주요 이유는 게임 파일의 오류 코드 및 버전명으로 추적할 수 있습니다. 예를 들어 "0123456789" 버전에서 특정 오류 코드가 발생하여 "서번트 클래스 X"와 "보구 Y" 설정이 삭제되었습니다.
- **오류 코드**: 실제 구체적인 맥락은 "EF9DCA" 오류 코드를 찾아내는데 집중해야 합니다. 이 코드는 해당 서번트 클래스와 보구 설정이 미사용된 지점을 알려주는 중요한 정보입니다.

#### 2. 비주얼 노벨 미공개 루트와 더미 데이터
- **비주얼 노벨 미공개 루트**: 주제에 대한 세부 질문으로, 독자가 실제 게임 파일에서 발견된 미사용 CG의 원래 분기점 추적이라는 과정을 이해하도록 합니다. 예를 들어 "이미지 A"와 "이미지 B"가 미사용 되었던 특정 맥락에서 서번트 클래스 "X"와 보구 "Y" 설정이 삭제되었음을 설명합니다.
- **더미 데이터**: 주제와 직접 관련된 선택 조건으로, 독자가 실제 게임 파일을 통해 미사용 CG의 원래 분기점 추적이라는 과정에 대해 자연스럽게 이해하도록 합니다. 예를 들어 "삭제된 서번트 클래스 X"와 "보구 Y" 설정이 삭제된 이유는 "EF9DCA 오류 코드"를 찾아내는데 집중되었음을 설명합니다.

#### 3. 에로게 성우 가명
- **에로게 성우 가명**: 실패 원인과 비교 기준으로, 독자가 실제 게임 파일에서 미사용 CG의 원래 분기점 추적이라는 과정을 이해하도록 합니다. 예를 들어 "이미지 A"와 "이미지 B"가 미사용 되었던 특정 맥락에서 서번트 클래스 "X"와 보구 "Y" 설정이 삭제되었음을 설명합니다.
- **비주얼 노벨 장르 설명**: 주제와 직접 관련된 비교 기준으로, 독자가 실제 게임 파일을 통해 미사용 CG의 원래 분기점 추적이라는 과정에 대해 자연스럽게 이해하도록 합니다. 예를 들어 "미사용 CG의 원래 분기점을 찾아내는 과정"과 "삭제된 서번트 클래스 X와 보구 Y 설정"이 삭제되었음을 설명합니다.

#### 4. 구체적인 맥락과 실패 원인
- **버전명 및 오류코드**: 실제 게임 파일에서 미사용 CG의 원래 분기점을 찾아내는 과정을 통해 버전명 "0123456789"와 오류 코드 "EF9DCA"가 구체적인 실패 원인으로 작용했습니다.
- **실제 데이터 활용**: 실제 게임 파일에서 미사용 CG의 원래 분기점 추적이라는 과정을 통해 버전명과 오류코드를 찾아내는 것이 중요합니다.
